# وثيقة الامتثال والأمان — Compliance & Security Overview
**آخر تحديث:** 2026-07-14 · تُطمئن العميل المؤسسي على ممارساتنا الفعلية (لا شهادة SOC2 رسمية بعد — انظر "الحدود" أدناه)

## 1) عزل بيانات العملاء (Multi-tenancy)
- كل عميل (مستأجر) معزول بالكامل عبر AsyncLocalStorage على مستوى كل طلب — لا استعلام يعبر حدود المستأجر.
- **مثبت باختبار آلي** يعمل مع كل تعديل كود (CI): `tests/tenant-isolation.test.js` — 5 سيناريوهات تمنع أي تسريب بين عميلين أو مع بيانات المشغّل.
- العميل الجديد يبدأ بقاعدة معزولة خاصة به مع شجرة حسابات IFRS كاملة (onboarding آلي).

## 2) التحكم بالوصول والهوية
- مصادقة JWT بانتهاء صلاحية 12 ساعة، كلمات المرور مخزنة bcrypt (لا نص صريح أبداً).
- قفل تصاعدي ضد تخمين كلمات المرور: 5 محاولات فاشلة → قفل 5 دقائق يتضاعف حتى 60.
- استرجاع كلمة المرور برابط بريدي موقّع صالح 30 دقيقة، أحادي الاستخدام، مسجّل في سجل التدقيق.
- أدوار وصلاحيات دقيقة لكل شاشة وفعل (مدير/محاسب/استقبال/مخزون/مشاهد).

## 3) سلامة البيانات المحاسبية (IFRS)
- القيد المزدوج مفروض برمجياً: يُرفض أي قيد غير متوازن (مدين ≠ دائن).
- قفل الفترات المحاسبية على التعديل والحذف (IAS 8).
- رواتب على حسابات منافع الموظفين الصحيحة (IAS 19) مع تصفية الالتزام عند الدفع.
- سجل تدقيق (auditLog) يوثّق العمليات الحساسة: من/متى/ماذا — بما فيها الإصلاحات الآلية مع نسخ احتياطية داخلية قبل أي حذف.
- القوائم المالية (دخل/ميزانية/تدفق) تتولد من القيود مباشرة — لا أرقام يدوية.

## 4) النسخ الاحتياطي والاستعادة
- نسخ كامل بأمر واحد (`npm run backup`) + زر تنزيل نسخة من داخل النظام (للمدير فقط).
- تدوير تلقائي (آخر 30 نسخة).
- **الاستعادة مُختبرة آلياً** في CI: دورة كاملة (نسخ → إتلاف → استعادة → تطابق بالحرف) في `tests/backup-restore.test.js`.
- MongoDB Atlas: تشفير أثناء التخزين (at-rest) والنقل (TLS) افتراضياً + نسخ Atlas السحابية (يُنصح بتفعيلها من لوحة Atlas).

## 5) مراقبة الأعطال
- كل خطأ خادم أو واجهة يُسجَّل داخلياً (آخر 200) ويُعرض للمدير عبر `/api/admin/errors` — نعرف بالمشكلة قبل اتصال العميل.
- وكيل "المُجرِّب" يفحص النظام كمحاسب حقيقي (57 شاشة + دورات محاسبية كاملة) ويصلح المشاكل الآمنة ذاتياً مع تقرير عربي.
- ميزانية أداء مراقبة: أي شاشة أبطأ من 1.5 ثانية تُعلَّم في تقرير الفحص.

## 6) خصوصية البيانات
- لا تُشارك بيانات العملاء مع أي طرف خارجي باستثناء Anthropic API (ميزات الذكاء الاصطناعي) — ولا تُرسل إليه بيانات تعريف المرضى.
- حذف بيانات عميل عند طلبه: متاح عبر المشغّل (حذف مستند المستأجر وقاعدة بياناته).

## 7) الحدود (بشفافية)
- لا شهادة SOC 2 / ISO 27001 رسمية بعد — هذه الوثيقة تصف الضوابط الفعلية المطبقة والمثبتة باختبارات.
- استرجاع كلمة المرور يتطلب تهيئة SMTP من الإعدادات (وإلا يُعاد التعيين عبر مدير النظام).
- المصادقة الثنائية (2FA) على خارطة الطريق.
